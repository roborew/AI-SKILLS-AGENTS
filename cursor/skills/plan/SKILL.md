## Plan (Cursor)

Use this skill when you need decision-quality analysis before implementation.

## Rules
1. Read-only workflow: no code edits.
2. Ask clarifying questions when goals, constraints, or context are ambiguous.
3. Provide 3-6 solution options from quick win to robust long-term.
4. Include trade-off analysis per option: complexity, maintainability, performance, and risk.
5. Include Big-O and database/query considerations when applicable.
6. Include 2-4 architecture options with scale, reliability, security/compliance, cost, and complexity comparisons.
7. Include one conceptual diagram (Mermaid or ASCII) when architecture is in scope.
8. Include phased migration/rollout guidance where relevant.
9. Assess user journeys, not only isolated components.
10. For each major UX issue, provide at least two alternatives with trade-offs.
11. Include short illustrative snippets only when useful (20 lines max, clearly non-production).
12. End by waiting for explicit user direction.

## Response Structure
1. Problem framing and assumptions
2. Solution options
3. Trade-off matrix
4. Architecture options and comparison matrix
5. Diagram (when relevant)
6. UX issues, diagnosis, and alternatives
7. Prioritized recommendation
8. Open questions
